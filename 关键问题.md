# Tarot Mirror - 关键问题与完善建议

## 🎯 产品定位与差异化

### Q1: 核心差异化是什么？
- 市面上已有不少塔罗应用（如Labyrinthos、Golden Thread Tarot等），我们的**核心差异化**是什么？
- 是"1分钟仪式"的极轻体验？还是"治愈任务联动"的闭环？还是"多占卜师陪伴感"？
- **建议**：明确1-2个核心差异化点，在UI/UX上重点突出

**✅ 已确认答案**：
**每日塔罗 + 月相综合分析**，结合轻疗愈，形成每日运势和建议，形成日历后有周总结和月总结。
- 这是我们的核心差异化点
- 结合塔罗牌和月相能量，提供更全面的每日运势分析
- 通过日历、周总结、月总结形成长期使用习惯和成长轨迹

### Q2: 目标用户更细分？
- 18-50岁范围较广，是否需要进一步细分？
- 例如：18-25岁（学生/初入职场）vs 30-50岁（职场/家庭压力）
- 不同年龄段的需求和付费意愿可能不同

## 🔮 塔罗内容与准确性

### Q3: 塔罗牌库与解读标准
- 使用哪套塔罗牌？（Rider-Waite、Thoth、其他？）
- 解读的准确性如何保证？是否有专业塔罗师审核内容？
- 78张牌的解读库如何建立？是AI生成还是人工编写？

### Q4: 多占卜师系统的实现
- "不同占卜师人设"具体如何实现？
  - 不同LLM模型（GPT-4、Claude、Gemini等）？
  - 不同prompt模板？
  - 不同解读风格（温柔/直接/神秘/理性）？
- 如何让用户感受到"陪伴感"而不是"机器感"？
- 占卜师是否需要固定形象/名字/背景故事？

**✅ 已确认答案**：
- **技术实现**：使用 DeepSeek、ChatGPT、Gemini 等 LLM，各自调用 API key
- **人设定义**：通过不同的 prompt 模板来定义不同塔罗师的人设和解读风格
- 每个 LLM 对应一个占卜师，通过 prompt 塑造独特风格

### Q5: 多占卜师对比的价值
- "同一牌阵多占卜师解读对比"是否会造成用户困惑？
- 如果解读差异很大，用户如何选择相信哪个？
- **建议**：考虑是否作为高级功能，或改为"换一个占卜师重新解读"

**✅ 已确认答案**：
- **第一版可以没有**多占卜师对比功能
- 后续版本可考虑添加此功能
