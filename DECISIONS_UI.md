# UI 设计决策记录

## ✅ 已确认的设计决策

### 1. 每日占卜触发时机
**决策日期**：2024-XX-XX

**决策内容**：
- 每天凌晨用户第一次进入 app，自动触发全屏占卜页面
- 右上角有跳过按钮，点击后直接进入主界面
- 主界面突出显示"今日占卜"按钮，引导用户完成每日占卜

**理由**：
- 保持"每日仪式"的连贯性
- 给用户选择权（可跳过）
- 通过引导按钮确保用户不会忘记占卜

---

### 2. LLM 调用时机
**决策日期**：2024-XX-XX

**决策内容**：
- 抽牌后立即调用 LLM API 生成内容
- 显示加载状态，等待生成完成

**理由**：
- 确保用户能看到实时生成的内容
- 避免主界面加载时的空白等待

---

### 3. 5个分类的生成方式
**决策日期**：2024-XX-XX

**决策内容**：
- 一次性生成所有5个分类的内容
- 用户点击分类按钮时直接加载已生成的内容

**理由**：
- 减少 API 调用次数，降低成本
- 提升用户体验（点击即显示，无需等待）

---

### 4. 分类按钮交互方式
**决策日期**：2024-XX-XX

**决策内容**：
- 展开/折叠方式，在同一页面内显示
- 点击分类按钮展开/折叠对应内容

**理由**：
- 保持用户在同一个页面，不打断浏览体验
- 交互简单直观

---

### 5. LLM Prompt 设计
**决策日期**：2024-XX-XX

**决策内容**：
- 尝试一次性生成所有内容（综合指引、具体占卜、分类占卜、幸运元素、疗愈任务）

**理由**：
- 减少 API 调用成本
- 确保内容的一致性
- 提升用户体验

**详细文档**：见 [`LLM_PROMPT_TEMPLATE.md`](./LLM_PROMPT_TEMPLATE.md)

### 6. 滑动条样式
**决策日期**：2024-XX-XX

**决策内容**：
- 分段滑动条，5个固定档位
- 滑动时有平滑动画

**理由**：
- 用户选择更明确
- 视觉反馈更清晰

---

### 7. 抽牌交互方式
**决策日期**：2024-XX-XX

**决策内容**：
- Demo 阶段同时支持长按和点击
- 长按更有仪式感，点击更快捷

**理由**：
- 给用户选择权
- Demo 阶段测试两种方式的用户偏好

---

### 8. 情绪状态的重要性
**决策日期**：2024-XX-XX

**决策内容**：
- **情绪状态是重要参数**，必须影响生成内容
- 内容要击中用户情绪，给予安慰或建议
- 针对不同情绪状态（特别是疲惫和焦虑），提供特别温柔和关怀的内容

**理由**：
- 这是"塔罗+轻疗愈"的核心差异化
- 个性化内容提升用户粘性
- 满足用户的情感需求

**具体策略**：
- 愉悦：鼓励利用积极能量
- 平静：加深平和感
- 疲惫：强调休息和自我照顾
- 迷茫：提供清晰方向
- 焦虑：给予安慰和接地气的建议

---

### 9. 疗愈任务
**决策日期**：2024-XX-XX

**决策内容**：
- 根据用户情绪状态生成，提供疗愈或缓解方法
- Demo 阶段由 LLM 生成
- 任务库可后续设计

**理由**：
- 情绪状态是核心，任务应该针对性生成
- Demo 阶段灵活度高
- 后续可以优化为任务库+LLM 推荐
